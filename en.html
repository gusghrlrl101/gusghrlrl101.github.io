Unmanned Store

  • Development for unmanned stores using real-time multi-camera person tracking and location estimation @ if Kakao presentation
    • DeepStream, TensorRT, Jetson, Gstreamer, C++, CUDA, Python, Influxdb, Kafka
  • The unmanned store project is a system that tracks and estimates the location of people in real-time through multiple cameras. Unmanned store services like Amazon Go mostly use 3D depth cameras, our goal was to implement it using a regular RGB camera. In Nvidia Jetson (edge device) environment, I used DeepStream based on C++ for video encoding and stream processing from multiple cameras. For real-time processing, inference were performed after converting to TensorRT, which has a fast deep learning model inference speed. In areas that need optimization, CUDA was used to improve performance. Gstreamer plugins for various functions were directly implemented and configured in the form of modules. InfluxDB and Kafka were utilized for time series data processing. I extended to cloud environment as well as edge device environment using Kubernetes. There were difficult moments as I went through design and development of the overall system structure all by myself, but I was able to successfully finish by setting long-term/short-term goals and processing them one by one.

Image Feature Search Engine PaaS

  • Development for PaaS using Argo Workflow for HNSW (Hierarchical Navigable Small World) and OPQ (Optimized Product Quantization) based search engine
    • Kubernetes, Argo Workflow, Python, Flutter
  • The image feature search engine is an Approximate KNN engine capable of indexing large amounts of image data. Data with large feature sizes are compressed using PQ and OPQ, sharded across multiple nodes, and indexed using HNSW. The image feature search engine is used in several services, such as recommending similar products in Shopping How. Processes such as data preprocessing, build and deployment have to be performed sequentially. Argo Workflow was used to configure the pipeline to make it easier to manage. In this way, I have organized the resources to be used more efficiently than in the conventional way. In addition, I developed an easy-to-use web environment so that service creation, data addition, and progress can be easily understood.

Traing Data Labeling Tool

  • Development for a web-based tool for labeling training data
    • Python, Flutter, Rust, Vue.js
  • When researching deep learning, it is necessary to label the data required for training. For this, I developed a web-based tool for image-related data labeling. The front-end used Flutter and Vue.js, and the back-end used Python and Rust. I have improved work efficiency by actively reflecting the users' requests for supplementary points and additional functions. In addition, I was able to understand the data flow while performing data collection and pre-processing tasks.
